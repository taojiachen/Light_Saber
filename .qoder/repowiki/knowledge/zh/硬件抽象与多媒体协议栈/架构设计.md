- **分层架构**：模块内部呈现清晰的“硬件-驱动-算法-应用”分层。`hardware_driver` 负责底层 I2S/Flash 物理接口；`IMU` 通过 `imu_port.h` 定义统一驱动接口，利用 CMake/Kconfig 实现 MPU6050/ICM20948 等传感器的编译期多态切换。
- **第三方库集成**：直接移植并封装了 Helix MP3、libogg 和 Opus (1.5.2) 的 C 语言源码，通过 `CMakeLists.txt` 精细控制源文件列表与编译宏（如定点运算、裁剪未使用代码），以适应嵌入式资源约束。
- **通信抽象**：`aliyun_mqtt` 和 `esp_websocket_client` 基于 ESP-IDF 原生网络栈进行二次封装，提供简化的事件驱动 API，屏蔽底层连接管理细节。
- **构建系统**：各子组件均包含独立的 `Kconfig` 和 `CMakeLists.txt`，支持通过 menuconfig 动态启用/禁用特定功能（如 Opus 编码器/解码器）或选择硬件型号。
